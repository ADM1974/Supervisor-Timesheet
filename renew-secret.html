Renewing the timesheet “security key”

For whoever maintains the J & D McLennan timesheet apps. Allow ~30 minutes.

What this is. The three timesheet apps (contractor, staff, supervisor) talk to SharePoint using a Microsoft client secret — a password that expires periodically. When it expires, every app stops being able to read or save timesheets (sign-ins may still work, but loading and submitting fail). This page is how you replace it. You do not need to touch the timesheet data — only the secret.

Step 1 — Create a new secret

  1. Go to entra.microsoft.com → App registrations → open “Contractor Timesheet”
    (Application/client ID cdf6e14d-b987-4bce-bfc3-8ea482b989b8 — this one registration's secret is shared by all three apps.)
  2. Left menu → Certificates & secrets.
  3. + New client secret → description e.g. timesheet-secret → expiry 24 months → Add.
  4. In the new row, copy the Value column (a long string like aB3Q~…) — NOT the “Secret ID” GUID. Copy it now; it's hidden once you leave the page.

Step 2 — Put the new secret on all three sites

In Netlify, for each site below: open it → Site configuration → Environment variables → edit CLIENT_SECRET → paste the new Value → Save.

Only CLIENT_SECRET changes. Leave every other variable as-is.

Step 3 — Redeploy all three sites

Netlify only picks up the new secret on a fresh deploy. For each site: Deploys → Trigger deploy → Deploy site. Wait for “Published”.

Step 4 — Delete the OLD secret

Do this last — only after all three sites are updated and redeployed. Entra → Contractor Timesheet → Certificates & secrets → delete the old (now-expired) secret row. If you delete it before updating a site, that site breaks until you fix it.

Step 5 — Clear this warning

The warning banner reads an expiry date in the supervisor app's config. Update it to the new secret's expiry:

  1. GitHub → ADM1974/Supervisor-Timesheet → open config.js → ✏️ edit.
  2. Change secretExpiry: "…" to the new secret's expiry date, format YYYY-MM-DD (e.g. a secret expiring 27 June 2029 → "2029-06-27").
  3. Commit changes. Netlify auto-deploys; the banner disappears.

How to check it worked

Open each app and sign in / enter a code. If timesheets load and save, the new secret is live. The supervisor app's warning banner should be gone once Step 5 deploys.

Quick summary: new secret → put its Value in CLIENT_SECRET on all 3 Netlify sites → redeploy all 3 → delete the old secret → update secretExpiry in the Supervisor-Timesheet config.js.
